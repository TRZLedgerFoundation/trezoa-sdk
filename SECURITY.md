# Security Policy

## Reporting Security Vulnerabilities

We take security seriously at Trezoa. If you discover a security vulnerability in the Trezoa SDK, please report it to us privately.

### How to Report

1. **Email**: Send details to security@trezoa.xyz
2. **Encrypted**: Use our PGP key for sensitive reports
3. **Bug Bounty**: Participate in our bug bounty program

### What to Include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

### Response Timeline

- **24 hours**: Initial acknowledgment
- **7 days**: Detailed response with assessment
- **30 days**: Fix deployed (for critical issues)

### Supported Versions

| Version | Supported |
| ------- | --------- |
| 3.x.x   | ✅ Yes    |
| 2.x.x   | ⚠️ Limited |
| 1.x.x   | ❌ No     |

## Security Best Practices

When using Trezoa SDK:

1. **Keep dependencies updated**
2. **Validate all inputs**
3. **Use secure key management**
4. **Audit your programs**
5. **Follow Trezoa security guidelines**

Thank you for helping keep Trezoa secure!
